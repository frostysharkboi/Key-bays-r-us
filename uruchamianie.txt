Biblioteki backend:
  Mariadb - host bazy Mysql
  Bodyparser - ułatwienie przetwarzania danych przesyłanych
  Cors - Cross-Origin Resource Sharing, w skrócie obsuga danych przesyanych i ich dostépów
Biblioteki frontend:
  Axios - obsługa połączenia z bazą
  Reacttable + Tanstack - formatowanie tabeli

w zewnętrznym folderze (server-test):

  pierwsze uruchomienie:
    terminal 1:
      cd user-service
      npm i express nodemon cors mariadb body-parser
      npm start
    terminal 2:
      cd user-web
      npm i @tanstack/react-table axios bootstrap @popperjs/core
      npm run dev

  każde kolejne uruchomienie:
    terminal 1:
      cd user-service
      npm start
    terminal 2:
      cd user-web
      npm run dev

By przygotować stronę do wrzucenia na serwer:
  w root.css zmienic:
    font-face {
      font-family: akira;
      src: url('src/assets/fonts/AODFont.otf');
    }
    na:
    font-face {
      font-family: akira;
      src: url('./assets/fonts/AODFont.otf');
    }
  cd user-web
  npm run build

By zaktualizować baze:
  wykonajcie w nowej konsoli kod python:
    cd steamdb-connection
    pip install -r requirements.txt
    python main.py

nie chcieliśmy cały czas łączyć się z bazą steam na dane gier szczególnie że wiele z nich wymaga dosyć ciężkiej edycji.
Znajduje się także kod pod dodawanie jednej gry po jej id w steam wykonywany tak:
  zmień wartość TARGET_APPID na chciany klucz gry (jest w linku strony danej gry na steam). póżniej w konsoli:
    cd steamdb-connection
    python import_single.py

By działać na lokalnej wersji bazy (z keybay.sql) trzeba ją zaimportować i zamienić komentarze w plikach:
  user-web/src/App.jsx (linijka 7 i 8)
    //export const axiosPath = "https://keysrus-backend.onrender.com";
    export const axiosPath = "http://localhost:3000";
  user-service/db/db.js (linijka 7 i 18)
